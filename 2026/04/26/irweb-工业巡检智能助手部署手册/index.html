irweb 工业巡检智能助手部署手册

JY2026-04-262026-04-26

记录 Smart Maintenance Copilot（智能维护助手）从本地 v0 项目到生产部署的完整流程，包括架构设计、子域名配置、视频流接入，以及后续后端接口规划。

项目概述

基于 v0 生成的 Next.js 16 + shadcn/ui 工业巡检仪表盘，三大面板：

  • 左侧 — 视频监控：实时 MJPEG 流 + 物体检测标注
  • 中央 — SOP 流程：标准作业步骤指引（可展开详情）
  • 右侧 — AI 助手：智能问答面板（当前模拟中）

后续后端接口规划

前端组件现已全部实现，但数据源均为硬编码。以下为接入真实后端需要的接口清单。

视频检测接口（VideoMonitor）

当前：MJPEG 流已就绪，检测框仍为模拟数据。

方法 路由 说明
WS /ws/detections WebSocket 推送检测结果 { id, label, status, x, y, width, height }[]

实现思路：Pi 端加一条线程跑轻量 YOLO，检测结果通过同一 WebSocket 推到前端。或者服务端用一个 Python 后端转发。

SOP 任务接口（SOPWorkflow）

当前：页面内 initialSteps 硬编码，状态不持久。

方法 路由 说明
GET /api/tasks 任务列表（含 SOP 步骤）
GET /api/tasks/:id 单个任务详情
POST /api/tasks 创建新任务
PUT /api/tasks/:id/steps/:stepId 更新步骤状态（pending→active→completed）

数据结构参考 sop-workflow.tsx:10-18 的 SOPStep 接口：

AI 助手接口（AIAssistant）

当前：setTimeout 模拟 1 秒后返回固定回复。

方法 路由 说明
POST /api/chat 发送消息，获取 AI 回复（建议 SSE 流式）
POST /api/chat/upload 上传图片供 AI 分析

请求体：

后端对接 Claude API / 通义千问等任意 LLM，用当前步骤上下文 + 历史消息构建 system prompt。

告警接口（AlertPopup）

当前：useEffect 5 秒后自动触发硬编码告警。

方法 路由 说明
WS /ws/alerts 服务端主动推送告警事件

方法 路由 说明
POST /api/auth/login 登录
GET /api/auth/session 当前会话
