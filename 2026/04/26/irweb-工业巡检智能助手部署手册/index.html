四、环境踩坑记录

4.1 DNS 修复

服务器使用 Tailscale DNS（100.100.100.100），无法解析外网域名。

修复：追加阿里云公共 DNS

4.2 pip 安装失败

Python 3.6.8 + 外网受限，pip 和 dnf 均无法正常使用。Tsighua/Aliyun 镜像也超时。

解决：放弃所有 pip 依赖，用 Python 3.6 标准库重写整个后端。

4.3 Nginx SSL 模块缺失

宝塔编译的 Nginx 未包含 ngx_http_ssl_module，但原配置中有 listen 443 ssl 指令。这导致 nginx -t 失败，无法 reload。

解决：移除整个 HTTPS server 块。ir.hwarkm.com 只用 HTTP 80 端口，HTTPS 由 CDN/反向代理层处理（如有需要）。

4.4 SSH 密钥配置

Windows 本地 SSH 密钥未在服务器 authorized_keys 中，无法免密登录。

修复（在宝塔终端执行）：

4.5 Python 3.6 兼容性

  • openai 包依赖 pandas>=1.2.3，后者需要 Python ≥ 3.7
  • fastapi 需要 Python ≥ 3.7
  • scikit-learn 1.3+ 需要 Python ≥ 3.8
  • 系统中同时有 Python 3.7（pyenv）和 Python 3.11（/usr/bin），但均为残损安装

最终方案：Python 3.6 标准库（http.server + urllib.request + json）零依赖实现全部功能。

十、LoongArch 平台迁移分析

10.1 前端

前端是 Next.js 静态导出（output: 'export'），产物为纯 HTML/CSS/JS 文件。

层面 是否需要改动 说明
部署方式 无 静态文件扔进 Nginx web root，与 CPU 架构无关
Nginx 配置 无 /api/* 和 /ws/* 代理规则完全一致
构建过程 无 在 x86 开发机上构建，产物直接 scp 到龙芯
CSS 渲染 无 浏览器端渲染，与服务器架构无关
JavaScript 无 浏览器端执行，与服务器架构无关
API_BASE 无 前后端同域部署，相对路径自动适配

结论：前端零改动。 唯一的”风险”是 Tailwind v4 + Turbopack 在龙芯上是否能跑 npx next build。但可以在 x86 上构建后部署，完全规避此问题。

10.2 后端

后端是单文件 Python 3.6 stdlib 程序。

层面 是否需要改动 说明
Python 解释器 无 银河麒麟 V10 自带 Python 3.6+
标准库依赖 无 http.server, json, urllib, re 全标准库
systemd 无 银河麒麟原生支持
外部 API 调用 待确认 DeepSeek API 需要外网访问；若龙芯云断网，需替换为本地 LLM
文件路径 微调 PDF_PATH 需改为龙芯上的绝对路径

结论：后端几乎零改动。 唯一需要确认的是龙芯云平台是否提供外网访问（调用 DeepSeek API）。如果不能，需在龙芯上部署本地大模型（如 Qwen），并修改 call_deepseek() 函数的 API_BASE 指向本地服务。

10.3 构建工具链

工具 x86 当前 LoongArch 替代
Node.js (前端构建) x86 上构建，不跑在龙芯上 不需要
Nginx x86 宝塔编译 yum install nginx（银河麒麟官方源）
systemd ✓ ✓（原生）
Python 3.6+ ✓ ✓（银河麒麟自带）

10.4 迁移清单

步骤 操作 风险
1 在 x86 上 npx next build 构建前端 无
2 scp out/ → 龙芯 /home/www/irweb/ 无
3 scp main_stdlib.py + image_index.json + images/ → 龙芯 /opt/irbackend/ 无
4 龙芯上 python3 --version 确认 ≥ 3.6 极低
5 龙芯上 python3 main_stdlib.py --port 8001 & 启动后端 低（确认 DNS 可用）
6 龙芯上配置 Nginx（复制现有 conf） 极低
7 龙芯上配置 systemd service（复制现有 unit） 极低
8 测试 curl http://localhost/api/chat 确认 DeepSeek API 可达

预计迁移耗时：30 分钟（不含龙芯环境准备）。
