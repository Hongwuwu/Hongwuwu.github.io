Linux Shell 脚本编程 —— 知识点速成

基于实验二（LINUX SHELL脚本程序设计）涉及的全部语法点整理，用于应对”查实验”环节的提问。
每个知识点按”是什么 → 实验里哪里用了 → 老师可能怎么问 → 你怎么答”组织。
就这个速成爽！早一天都不会看的！

一、脚本基础结构

1. Shebang（#!/bin/bash）

是什么： 脚本文件的第一行，告诉操作系统用哪个解释器执行这个脚本。#! 是固定格式，后面跟解释器的绝对路径。

实验哪里： menu_ 第1行、original.sh 和 modified.sh 都以此开头。

可能问： “#!/bin/bash 可以省略吗？”

怎么答： 如果用 bash 文件名 的方式执行可以省略，因为此时我们手动指定了解释器。但如果用 ./文件名 直接执行，必须有这一行，否则系统不知道该用什么解释器，默认会用 /bin/sh，可能与 bash 语法不完全兼容。

2. chmod +x（赋予执行权限）

是什么： 给文件添加”可执行”属性。Linux 中每个文件都有 r(读) w(写) x(执行) 三种权限。

实验哪里： chmod +x menu_、chmod +x original.sh、chmod +x modified.sh

可能问： “为什么要 chmod +x？不执行这步会怎样？”

怎么答： 新建的脚本文件默认没有执行权限。如果直接 ./menu_ 会报 “Permission denied”。chmod +x 就是告诉系统”这个文件可以当作程序运行”。也可以用 bash menu_ 绕过，但最好的做法还是加权限。

3. 注释（#）

是什么： # 开头到行尾的内容被Shell忽略，用于写说明。唯一的例外是第一行的 #!。

实验哪里： #menu、#loop forever!、# 原始功能 等

可能问： “#! 和 # 有什么区别？”

怎么答： #! 在第一行且连在一起（没有空格）是特殊的，表示解释器路径。单独的 # 就是普通注释，Shell 直接跳过不执行。

二、变量

4. 变量定义与引用

是什么： Shell变量不需要声明类型，直接 变量名=值 即可定义。引用变量时前面加 $，如 $NAME 或 ${NAME}。

实验哪里： NAME="杨蕙葳"、MYDATE=\date +%d/%m/%Y`、CHOICE、FILENAME` 等

铁律：等号两侧绝对不能有空格！

可能问： “为什么你定义变量时等号两边没空格？”

怎么答： Shell语法强制要求。有空格的话Shell会把变量名当成命令去执行。

5. 命令替换（反引号 vs $()）

是什么： 把一条命令的输出结果赋给变量。两种写法等价：

实验哪里： menu_ 中用反引号获取日期和主机名；其他地方多用 $()

可能问： “反引号和 $() 有什么区别？”

怎么答： 功能完全一样。反引号是老写法，$() 是新写法，可读性更好、可以嵌套。menu_ 骨架是老师给的，用了反引号；我自己写的部分倾向于用 $()。

6. 引号的区别（双引号 vs 单引号 vs 无引号）

写法 效果
"$var" 变量替换和转义字符（\t, \n）生效
'$var' 一切原样输出，不替换变量，不处理转义
$var 变量会替换，但空格会导致单词拆分

实验哪里： "$FILENAME"（双引号，变量能替换）、"条件为真"（纯粹字符串）

可能问： “为什么有些变量加了双引号，有些没加？”

怎么答： 加双引号是安全做法。比如 [ "$xb" = "female" ]，如果 $xb 是空值，不加引号就变成 [ = "female" ]，语法报错。加上引号后变成 [ "" = "female" ]，虽然不相等但语法正确。养成给变量加双引号的习惯可以避免很多坑。

三、输入输出

7. echo 命令（输出到屏幕）

是什么： 在终端显示文字。

选项 效果 示例
echo "hello" 输出 hello，末尾自动换行 —
echo -n "hello" 输出 hello，不换行（no newline） 提示后光标停在同行
echo -e "a\tb" \t 被解释为Tab（enable escapes） —

实验哪里： echo -e -n "\tYour Choice..." —— 既要Tab缩进（-e）又不想换行（-n）

可能问： “-e 和 -n 分别干什么的？”

怎么答： -e 是让 \t、\n 这些转义字符生效（不然会被当普通文本输出）。-n 是输出后不换行，光标停在当前行末尾，方便用户在同行输入。

8. read 命令（读取用户输入）

是什么： 从键盘读取一行输入，存入指定变量。

实验哪里：

  • read CHOICE —— 读菜单选项
  • read FILENAME —— 读文件名
  • read INDATE —— 读日期
  • read DUMMY —— 读一个”废”变量，实现暂停

可能问： “read DUMMY 这个变量名是什么意思？为什么用完后不用它？”

怎么答： DUMMY 就是”哑变量”，它接收到的值（用户按的回车）我们不在乎，后面也不引用它。这里的唯一目的就是让程序暂停，等用户看完输出按回车后再刷新菜单。相当于”按任意键继续”。

可能问： “read 后面可以跟多个变量吗？”

怎么答： 可以。比如 while read xh bj xb ma ch en 就是一次读取一行，按 IFS 切分成6段，分别赋给6个变量。

9. echo -e 转义字符

是什么： echo -e 后，\t 变成Tab、\n 变成换行、\007 变成蜂鸣声。

实验哪里：

  • echo -e "$xh\t$bj\t$ma\t$ch\t$en" → 用Tab分隔各字段
  • echo -e "\t\007 error" → Tab缩进 + 蜂鸣警告

可能问： “\007 是什么意思？”

怎么答： \007 是ASCII码中的 BEL（Bell）字符，发送给终端后会让终端发出”嘀”的一声，用来提醒用户输入了无效选项。

10. Heredoc（cat <<MARKER ... MARKER）

是什么： 把多行文本作为一个整体输出，不用写很多echo。

以 MAYDAY 作为开始和结束标记，中间所有内容原样输出。

实验哪里： menu_ 中用 cat <<MAYDAY ... MAYDAY 输出整个菜单界面

可能问： “MAYDAY 是什么关键字？能换别的吗？”

怎么答： MAYDAY 不是关键字，只是我选的一个结束标记。可以是任意字符串，只要开头和结尾一致就行。常见的有 EOF、END、HELP 等。

11. 输出重定向（> 和 >>）

符号 效果
> file 把输出覆盖写入file（file不存在则创建）
>> file 把输出追加到file末尾（保留原内容）
2>/dev/null 把错误输出扔到”黑洞设备”

实验哪里：

  • >nf.txt —— 清空 nf.txt，确保每次运行从头开始
  • >>nf.txt —— 追加符合条件的记录
  • echo "2024011456" >> sc2 —— 追加个人学号到文件末尾
  • 2>/dev/null —— 把错误输出扔到”黑洞设备”

可能问： “>nf.txt 前面没有命令，是什么语法？”

怎么答： 单独一个 > 后面跟文件名，效果就是清空/创建该文件。等价于把”空”重定向进去。目的是确保每次运行脚本时输出文件都是干净的，不会残留上次运行的数据。

可能问： “2>/dev/null 的 2 和 /dev/null 分别是什么意思？”

怎么答： 每个进程有三个标准流：0=标准输入，1=标准输出，2=标准错误。2>/dev/null 就是把错误信息（流2）重定向到 /dev/null。/dev/null 是Linux的”黑洞设备”，写进去的数据直接消失。在脚本中这样做是为了”静默”——我们只关心 df 命令是否成功运行，不想让”分区不存在”的报错显示在屏幕上。

四、流程控制

12. while 循环

是什么： 条件为真时反复执行循环体。

实验哪里：

  • while true ... done —— 条件永远是true，无限循环，只有exit才能跳出
  • while read xh bj ... done <pc1.txt —— 逐行读文件，读完自动结束

可能问： “while true 不会死循环吗？怎么退出？”

怎么答： 就是要死循环。菜单程序需要反复显示、反复接受用户输入，不能执行一次就退出。退出靠用户在菜单中选 Q，执行 exit 0 终止整个脚本。

可能问： “while read 什么时候结束？”

怎么答： read 每次读一行，读完最后一行后再读就会失败（返回非零退出码），while 检测到条件为假，循环自动终止。所以它恰好读完整个文件，不会多读也不会少读。

13. case 语句（分支选择）

是什么： 根据变量的值走不同分支，类似其他语言的 switch。

实验哪里： menu_ 中根据 CHOICE 分发到 A/B/C/Q/* 五个分支

重点语法：

  • A|a) 表示匹配 “A” 或 “a”，实现大小写兼容
  • 每个分支以 ;; 结束（相当于 break）
  • esac 是 case 倒过来写，表示 case 块结束

可能问： “A|a 里面的 | 是什么意思？”

怎么答： | 是”或”的意思，A|a 表示用户输入大写A或小写a都匹配这个分支。这样用户不用关心大小写。

可能问： “为什么最后有个 *) ？”

怎么答： * 是通配符，匹配任意字符串。当前面 A/B/C/Q 都没匹配到时，就落进 *) 分支，相当于”其他所有情况”。这里是用来处理用户按了无效选项的情形，给出错误提示。

14. if 条件判断

是什么：

[ ] 本质上是一个叫 test 的命令，[ 是它的别名。

实验哪里：

  • if [ -e "$FILENAME" ] → 判断文件是否存在
  • if [ $? -eq 0 ] → 判断上一条命令是否成功
  • if [ -n "$INDATE" ] → 判断字符串是否非空
  • if [ "$xb" = "female" -a ... ] → 复合条件判断

[ ] 里面的空白是强制的：[空格"$xb"空格=空格"female"空格] — 没有空格会报错！

可能问： “if 和 [ 之间为什么一定要有空格？”

怎么答： 因为 [ 本身是一个命令（就是 test 命令），不是语法关键字。命令和参数之间必须有空格。写 if["$x"="y"] 相当于执行一个叫 if[xxx] 的命令，当然找不到。

15. 文件测试与字符串测试

常用文件测试（实验里用到的）：

写法 含义
[ -e file ] 文件是否存在（exist）
[ -f file ] 是否为普通文件
[ -d dir ] 是否为目录

常用字符串测试：

写法 含义
[ -z "$s" ] 字符串是否为空（zero length）
[ -n "$s" ] 字符串是否非空（non-zero）
[ "$a" = "$b" ] 字符串相等
[ "$a" != "$b" ] 字符串不等

实验哪里： [ -e "$FILENAME" ] 检查文件在不在；[ -n "$INDATE" ] 检查用户有没有输入日期（空输入就跳过查询）。

16. 数值比较 vs 字符串比较

这是一个极其重要的区分点，老师大概率会问！

数值比较（用于数字）：

写法 含义
-eq 等于（equal）
-ne 不等于（not equal）
-lt 小于（less than）
-le 小于等于（less or equal）
-gt 大于（greater than）
-ge 大于等于（greater or equal）

字符串比较（用于文本）：

写法 含义
= 相等
!= 不等

实验哪里：

  • [ $ch -lt 60 ] ← ch 是数字（成绩），用 -lt
  • [ "$xb" = "female" ] ← xb 是字符串（性别），用 =

可能问： “为什么比较成绩用 -lt 而不是 < ？”

怎么答： 在 [ ] 内部，< 会被解释为输入重定向，不是”小于”。数值比较必须用 -lt、-gt 这些字母缩写。

可能问： “为什么 $ma -ge 80 不用加引号，而 $xb 加了引号？”

怎么答： 数字变量不加引号也可以（因为数字不会含空格或为空），但文字变量可能为空，不加引号会导致语法错误。统一都加引号是最好的。

17. 逻辑操作符（-a、-o、&&、||）

在 [ ] 内部（test命令内部）：

  • -a → AND（与）
  • -o → OR（或）

在 [ ] 外部（Shell命令级别）：

  • && → 前一条命令成功才执行后一条
  • || → 前一条命令失败才执行后一条

实验哪里：

  • [ $ch -lt 60 -o $en -lt 60 -o $ma -lt 60 ] → 三个条件”或”
  • [ $xb = "female" -a $ma -ge 80 -a $ch -ge 80 ] → 三个条件”与”
  • [ 条件 ] && let count++ → 条件为真才自增

可能问： “-a 和 && 有什么区别？什么时候用哪个？”

怎么答： -a 用在 [ ] 内部，连接多个测试条件。&& 用在 [ ] 外部，连接两条独立的命令。

最后这种写法是 && 的典型用法：短路的含义是”只有前面成功了，后面才值得做”。

五、数据处理核心

18. IFS（内部字段分隔符）

是什么： Internal Field Separator，Shell读取数据时按什么字符切分字段。默认值是空格、Tab、换行符。

实验哪里： IFS="," 将分隔符改为逗号，使read能正确解析pc1.txt的CSV格式。

可能问： “为什么改 IFS？不改会怎样？”

怎么答： pc1.txt 的数据是逗号分隔的（S001,C03,female,52,83,71）。不改 IFS 的话，read 会按空格/Tab切分，一整行被当成一个字段，六个变量中只有 xh 拿到整行数据，其余五个全为空。

可能问： “为什么先 oldifs="$IFS" 存起来，最后再 IFS=$oldifs？”

怎么答： 改 IFS 会影响当前Shell会话的所有后续操作。先备份再恢复是一种防御性编程，避免脚本跑完后你的终端出现奇怪行为（比如命令参数按逗号切分而非空格切分）。

19. while read 逐行处理文件

是什么：

这是Shell处理文本文件的标准模式，逐行读取、逐行处理。

实验哪里： while read xh bj xb ma ch en; do ... done <pc1.txt

可能问： “为什么要用 while read 而不是 cat pc1.txt？”

怎么答： while read 是逐行处理，每次只占一行内存，适合大文件。cat 一次性读入整个文件。对于100行的小文件两者都可以，但 while read 是Shell文本处理的标准范式。

可能问： “<pc1.txt 放在 done 后面是什么意思？”

怎么答： 这叫”输入重定向到循环”，把pc1.txt的内容作为整个 while 循环的标准输入。这样循环内部的 read 每次从文件读一行，而不是从键盘读。

20. let 命令和算术运算

是什么： Shell中做整数运算的几种写法：

实验哪里： let count++、let count_c03++、total=$((ma + ch + en))

可能问： “let 和 $(( )) 有什么区别？”

怎么答： let 是一个独立命令，$(( )) 是算术扩展，可以嵌套在字符串中。推荐用 $(( ))。

21. 管道（|）

是什么： 把左边命令的输出作为右边命令的输入。

实验哪里：

  • df -h "$DISK" | tail -1 → df输出传给tail取最后一行
  • df ... | tail -1 | while read ... → 多级管道

核心概念——子Shell： 管道右侧的命令运行在子Shell中，子Shell中的变量修改不影响父Shell。

可能问： “管道和重定向有什么区别？”

怎么答： 管道 | 连接两个命令（命令A的输出→命令B的输入）。重定向 > / >> / < 连接命令和文件。

六、关键命令详解

22. stat 命令（文件信息查询）

是什么： 显示文件或文件系统的详细信息。

实验哪里： 选项A，查询文件的大小、权限、创建/修改时间。

选项 含义
stat -c %s 文件 文件大小（字节数）
stat -c %A 文件 权限字符串（如 -rw-r--r--）
stat -c %w 文件 创建时间 / Birth time
stat -c %y 文件 最后修改时间

可能问： “为什么要用 stat 而不是 ls -l？”

怎么答： ls -l 只能看到大小和权限，看不到创建时间。stat 信息更全面，而且可以用 -c 精确控制输出格式，方便脚本解析。

23. df 命令（磁盘空间查询）

是什么： 显示文件系统的磁盘使用情况。

实验哪里： 选项B。

选项 效果
df 所有挂载分区的使用情况
df -h 人类可读格式（KB/MB/GB）
df /dev/sda1 仅查询指定分区

可能问： “为什么用 df -h 而不是 df？”

怎么答： df 默认以字节或块为单位，数字很大不直观。-h（human-readable）自动换算为 1.2G、500M 这类单位，一目了然。

24. date 命令（日期时间）

是什么： 显示或设置系统日期时间。

实验哪里： 选项C + 菜单标题栏的日期显示。

常用格式：

格式 输出示例
date +%d/%m/%Y 27/05/2026
date +%A Wednesday
date +%j 147（一年中的第几天）
date +%Y-%m-%d 2026-05-27

date -d 的妙用：

可能问： “date -d 是什么意思？”

怎么答： -d 选项允许我们解析一个日期字符串（而非只能用当前时间），实现对任意日期的查询。这个参数是GNU date的特性，在Linux上可用。

25. 退出状态码 $?

是什么： 每条命令执行完后都有一个退出状态码（0~255）。0 = 成功，非0 = 各种错误。$? 保存了上一条命令的退出状态码。

实验哪里：

  • if [ $? -eq 0 ] → 判断 date -d 是否成功解析了用户输入的日期
  • df -h /dev/sda1 >/dev/null 2>&1 → 我们不看输出，只看 $? 是否为0

可能问： “$? 是什么意思？值从哪来的？”

怎么答： $? 是一个特殊变量，始终保存上一条命令的退出码。0表示成功，非0表示失败。可以用来判断前一条命令是否按预期执行了。

26. /dev/null（黑洞设备）

是什么： Linux系统中的一个特殊文件，所有写入它的数据都会被丢弃。

实验哪里： df -h /dev/sda1 >/dev/null 2>&1

  • >/dev/null → 丢弃正常输出
  • 2>&1 → 把错误输出也转到同一个地方

这条命令我们不在意 df 显示什么，只关心它有没有报错（通过 $? 判断）。

可能问： “>/dev/null 2>&1 这一串到底啥意思？”

怎么答： 把正常输出（1）和错误输出（2）全都扔进垃圾桶。我们的目的不是看df的内容，而是探测 /dev/sda1 这个分区是否存在。探测成功 → $?=0，探测失败 → $?≠0。

七、老师常问的高频问题（汇总）

Q: “你这个菜单程序的关键结构是什么？”

A: while true 无限循环 + case 分支。循环保证菜单反复显示，case 根据用户选择分发到不同功能。只有 Q 分支调用 exit 才能终止循环。

Q: “为什么用 while true 而不是 for 循环？”

A: for 循环需要一个确定的列表（比如 for i in 1 2 3）。菜单程序不知道用户要用多少次、什么时候退出，所以用 while true 无限循环更合适。

Q: “read 和 echo 配合是怎么实现交互的？”

A: echo 输出提示信息 → read 等待用户输入并存入变量 → 脚本根据变量值做分支判断。这是Shell交互的基本模式。

Q: “IFS 到底影响什么？”

A: 影响 read 和 for 等命令如何把一行文字拆成多个字段。IFS 中的字符被当做分隔符。数据是逗号分隔的，就得把 IFS 设为逗号。

Q: “Shell变量为什么要加双引号？”

A: 防止变量值为空或被空格拆分时产生语法错误。比如 [ $x = "abc" ]，如果 x 为空，变成 [ = "abc" ]，Shell 报语法错误。[ "$x" = "abc" ] 即使 x 为空也是 [ "" = "abc" ]，语法正确。

Q: “你的脚本里用到的 $(( )) 是什么意思？”

A: 算术扩展，计算括号内表达式的结果。total=$((ma + ch + en)) 把三科成绩加起来赋给 total。Shell 默认把所有变量当字符串，$(( )) 告诉它”这里面是整数运算”。

Q: “如果文件不存在，你的脚本怎么处理？”

A: [ -e "$FILENAME" ] 先检查文件是否存在，存在才执行 stat 查询，不存在则输出”文件不存在”的错误提示。这叫”防御性检查”，避免对不存在的文件执行无意义的操作。

Q: “怎么保证脚本多次运行不会数据叠加？”

A: 在循环前用 >nf.txt、>sc1、>sc2 清空输出文件，这样每次运行都是从干净状态开始。如果不清空，>> 追加就会把新旧数据混在一起。

Q: “短路求值 && 和 || 是怎么工作的？”

A:

类似其他语言的 if (条件) 命令; 但更简洁。实验里用它做”条件计数”：[ 不及格 ] && let count++。
